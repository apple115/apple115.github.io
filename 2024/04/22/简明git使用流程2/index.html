简单git 使用方法

为什么害怕使用一个自己不知道的东西，最大的问题不知道如何撤销这个
所以这次的使用会从 使用这个命令 如何使这个命令返回成原来的状态，同时模拟 git 冲突 实践这个
目的是使用一个不会使用git的的人，快速可以使用git 与人合作

git 初始化

git init 使一个文件夹 初始化

取消这个git init如何不小心在一个文件中使用了这个git init 如何取消呢

当我们使用这个git init初始化的时候
会在当前目录下有一个.git 文件夹,删除他就可以了

查看git 状态

使用git status,这个命令不会改变当前目录下的文件

使用git 跟踪文件

添加一个1.c文件
使用git status查看这

发现当前文件夹git 没有跟踪这个文件

我们使用git add <这个文件>
将这个文件跟踪添加进去

如过我不小心添加来这个文件,想取消这个添加可以使用git reset <这个文件>

使用git 发起一次提交,在git add 添加的情况下

使用git进行一次提交git commit -m "<提交的信息>"

这样就算一次提交

如何我认为这个提交有问题，如何取消呢
有两种情况
第一：我的commit的信息有问题，我只是想修改一下信息
使用git commit --amend
进入一个界面修改
在出来就可以

第二：我想撤销这个commit,不想保留
使用git reset --soft HEAD^

第三：保留这个commit，但我想回到我想回到的状态

你会保留这个但可以回到你想回到的地方

可以使用git log 查看这个历史

git log 查看这个历史
